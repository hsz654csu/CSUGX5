# Architecture

The Simulink model is generated by `scripts/create_course_model.m` and saved as `model/uav_6dof_course.slx`.

## Top-Level Modules

| Module | Role |
|---|---|
| `Guidance` | Generates reference position, velocity, acceleration, and yaw commands. |
| `Navigation Feedback` | Provides delayed state feedback to avoid algebraic loops. |
| `Position Controller` | Converts position tracking error into total thrust, attitude reference, and acceleration command. |
| `Attitude Controller` | Converts attitude error into roll, pitch, and yaw torque commands. |
| `Control Allocation` | Maps total thrust and body torques to four motor speed commands. |
| `Propulsion Dynamics` | Simulates first-order motor/rotor speed response. |
| `6DOF Dynamics` | Updates translational and rotational states from rotor thrust and torques. |
| `Data Logging` | Saves reference, control, rotor speed, state, and acceleration data. |

## State Vector

```text
state = [x, y, z, vx, vy, vz, phi, theta, psi, p, q, r]^T
```

where position is in meters, velocity is in meters per second, Euler angles are in radians, and body rates are in radians per second.

## Reference Trajectory Vector

```text
ref_traj = [x_ref, y_ref, z_ref, vx_ref, vy_ref, vz_ref, ax_ref, ay_ref, az_ref]^T
```

The guidance module uses quintic polynomial segments so that position, velocity, and acceleration are continuous at segment boundaries.

## Control Chain

```text
ref_traj + nav_state
    -> position controller
    -> U1_cmd, att_ref, acc_cmd
    -> attitude controller
    -> [U1, tau_phi, tau_theta, tau_psi]
    -> control allocation
    -> motor_cmd
    -> propulsion dynamics
    -> rotor_speed
    -> 6DOF dynamics
    -> state
```

## Dynamics Scope

The current model uses simplified rotor-speed-driven rigid-body dynamics:

- rotor thrust: `T_i = kf * w_i^2`
- total thrust: `U1 = sum(T_i)`
- body torques from differential rotor speeds
- gravity
- linear translational damping
- angular damping
- Euler-angle kinematics

This is suitable for educational closed-loop simulation and architecture validation. It is not intended to replace high-fidelity flight dynamics or aerodynamic simulation.
